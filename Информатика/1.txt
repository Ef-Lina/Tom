1. Git — это система для управления версиями исходного кода программ. Проект был создан Линусом Торвальдсом для управления разработкой ядра Linux, первая версия выпущена 7 апреля 2005 года. Система контроля версий страхует от ошибок и возвращает код в то состояние, когда всё работало.
Git — это система контроля версий, а GitHub — онлайн-хранилище кода.
2. Система контроля версий позволяет не ждать обновления главной ветки и разрешает всем участникам команды свободно перемещаться между ветками других разработчиков для копирования нужных фрагментов кода. Так же это позволяет корректно объединить результаты нескольких разработчиков, предотвращая конфликты.
3. Основные термины:
	Коммиты - контрольные точки кода. Один коммит — это пакет изменений, хранящий информацию с добавленными, отредактированными или удалёнными файлами кода.
	Master-ветке - основная версия проекта, которая после завершения работы превратится в продукт. На ней расположены коммиты.
	Ветка - это подвижный указатель на один из коммитов.
4. Создавать различные ветки проекта необходимо, если необходимо исправить ошибки в исходном коде, но программа должна продолжать работать; одновременно необходимо добавить несколько обновлений; над проектом работают несколько независимых групп.
5. Merge позволяет произвести слияние веток с учётом времени изменения в них, при этом теряется логика каждого отдельного кода. Rebase позволяет сохранить идею каждой разработки, при этом может быть утерена последовательность разработок(история).
6. Централизованные системы контроля версий используют один общий сервер, взаимодействие всех пользователей происходит только через него. Вся информация всегда актуальна. Любой сбой на сервере несёт за собой полную потерю информации. В случае же децентрализованных систем контроля версий кроме удалённого репозитория у каждого пользователя есть свой локальный репозиторий. Есть определённые проблемы с актуальности информации на удалённом репозитории. В сучае проблем с файлами на основном репозитории всё можно восстановить, используя данные локальных репозиториев.
